# Parameters

### Default Parameters

Parameters are named variables, passed into a function, and are used to **import** the arguments into functions. 

Effectively, arguments are **realized** parameters through a function call.

An example:

    src="https://hlab.app/s/docs/paramex1_hbook"
    width="100%"
    height="500px"/>


Here are some key differences between parameters and arguments:


- Function parameters are the names listed in the function's definitions. (parameter1 in our case above)

- Function arguments are the actual values passed to the function. 
    - (variable argument1 (const 50) is a parameter that becomes an argument)

- Arguments are set to the values of the parameters supplied. 

:::info

The fundamental, basic idea of parameters are that they are unrealized arguments.

Arguments can't exist without parameters.

:::

An important feature regarding parameters is the 2 versions: 

- Input parameters (most common kind; pass values into functions)

- Output/return parameters (primarily return multiple values from a function)

    - Not recommended as they can cause confusion

### Rest Parameters

What's the difference between **default parameter** and a **rest parameter**?

- **Default parameters** allow named parameters to be initialized with default values

    - Used when certain parameters are optional (so if they don't put one in it becomes the default value)

- **Rest parameters** allow an indefinite, probably unnamed, amount of arguments as an `Array`.

    -  It creates certain flexibility in function definitions.

Only one rest parameter is allowed in a function, and it must be the last parameter in the definition.

There are a few technical differences between rest parameters and the argument object which should be mentioned:


- The arguments object is not a real `Array`, while rest parameters are `Array` instances.

- The arguments object has additional functionality specific to itself like the callee property for example. 

- The `...restParam` bundles all extra parameters into one array.

    - On the other hand, arguments object contains ALL the parameters - including the parameters in `...restParam`, all in one.


:::info

Access the length of restParams easily: `{restParamName}.length;`

:::
