# Parameters - Default and Rest Parameters

### Default Parameters

Parameters are named variables, passed into a function, and are used to **import** the arguments into functions. 

Effectively, arguments are **realized** parameters through a function call.

A simple example:

Here are some key differences between parameters and arguments:


- Function parameters are the names listed in the function's definitions. (parameter1 in our case above)

- Function arguments are the actual values passed to the function. (argument1 (const 50) is a parameter that becomes an argument)

- Arguments are set to the values of the parameters supplied. 

:::info

The fundamental, basic gist of Parameters are that they are unrealized arguments. Placeholders, transformers. Arguments can't exist without Parameters.

:::

An important feature regarding parameters is their 2 versions: Input parameters (most common kind; pass values into functions), and output/return parameters (primarily return multiple values from a function - these are not recommended as they cause confusion).

An example is not important as it can cause confusion.

Let's move on to the difference between a **default parameter** and a **rest parameter**:

- **Default parameters** allow named parameters to be initialized with default values if no value (or `undefined`) is passed to the function.
    - Used when one isn't sure if the user will pass a value for a certain parameter or more commonly - use it when certain parameters are optional (so if they don't put one in it falls down to a default value)

- **Rest parameters** allow an indefinite, probably unnamed, amount of arguments as an `Array`. It allows certain flexibility in function definitions.

:::tip

Only 1 rest parameter is allowed in a function, and it must be the last parameter in the definition.

:::

There are a few technical differences between rest parameters and the argument object which should be mentioned:


- The arguments object is not a real `Array`, while rest parameters **are** `Array` instances.

- The arguments object has additional functionality specific to itself like the callee property for example. 

- The `...restParam` bundles all extra parameters into one array, therefore it doesn't contain any named argument defined before the `...restParam`. 
    - On the other hand, arguments object contains ALL the parameters - including the parameters in `...restParam` `Array` - bundled into 1 array-like object.


:::note

Access the length of restParams easily: `function funcName(...restParams1) { print(restParams1.length); }`

:::

Overall, learning to use rest parameters in combination with default parameters allows for the most flexible, powerful and specific function definitions.
