# Parameters - Default and Rest Parameters

### Default Parameters

Parameters are a concept one may be familiar with. Parameters are, in a function call, the function's arguments. Specifically it is a named variable passed into a function, and are used to **import** the arguments into functions. 

A simple example:

```js
function myFunction(parameter1) {
    print(parameter1);
}
// this function's parameters are: parameter1, and it'll become an argument when a value is passed in:

const argument1 = 50;

myFunction(argument1);

//So now the function gets passed an argument called 'argument1' i.e. it gets passed the const 50.
//the parameter(s) let the function perform operations on the argument, in this case it simply prints it.
//Hence the output will simply be "50".
```

Here are some key differences between parameters and arguments:

<ul>
<li>Function parameters are the names listed in the function's definitions. (parameter1 in our case above)</li>
<li>Function arguments are the actual values passed to the function. (argument1 which is a const of 50 in the example above)</li>
<li>Parameters are initialized to the values of the arguments supplied. </li>
</ul>

An important feature is that there are two different types of parameters: Input parameters (most common kind; they pass values into functions), and output/return parameters (primarily return multiple values from a function - these are not recommended as they cause confusion).

Now let's discuss the difference between a **default parameter** and a **rest parameter**:

Default parameters allow named parameters to be initialized with default values if no value (or undefined) is passed to the function:

```js
function binaryOperation1(a, b=1) {
    return 2*a + 3*b;
}
print(binaryOperation1(4,2));
// what will happen is the default parameter value for b is overridden with the supplied value, 2, so we get 2*4 + 3*2 = 8+6 = 14 as output
print(binaryOperation1(4));
// in this case no value is passed for the parameter 'b'. Hence it takes the default value of '1'. So we get 2*4 + 3*1 = 8 + 3 = 11 as output
```

Use default parameters when you aren't sure if the user will pass a value for a certain parameter, or most commonly - use it when certain parameters are optional. So if the user wants to use the function without the optional parameter, the default value will take hold and you can control the function that way.

### Rest Parameters

The rest parameter syntax allows a function to accept an indefinite number of arguments as an array, which is commonly used and quite important. It allows quite a bit of flexibility in function definitions. Here is an example of how it's used:

```js
function sum(...theArgs) {
    let result = 0;
    for (const arg of theArgs) {
        result = result + arg;
        //another way to write the above is: result += arg;
    }
    return result;
}

print(sum(1,2,4)); //creates a 3 indice array arguments object containing 1,2, and 4 in that order. Then loops through as the function says and adds them: 1+2+4 = 7.
print(sum(1,1,1,1,5)); //creates a 5 indice arguments object array with values 1,1,1,1,5. The output will be 1+1+1+1+5 = 9.
print(sum(10)); //simply runs the function with the sole argument '10' returning 10 as nothing is summed to it.
```
One can also mix normal **and** default parameters with rest parameters, for very flexible function definitions:

```js
function myFunction(a,b=3, ...otherArgs) {
    print("a: "+a);
    print("b: "+b);
    print("other arguments: "+otherArgs);
}

myFunction(1,2,3,4,5,6);

//the output will be this: a: 1 b: 2 other arguments: [3, 4, 5, 6]
//that is because a is set to be the first argument, 1 so it will print: a: 1 and similiarly for b, 2 overwrites the default value so it prints b: 2. And it lastly prints the otherArgs in an array: other arguments: [3,4,5,6].

myFunction(1);
//this will make a=1, since b isn't given it takes the default value of 3 and since the number of extra arguments is 0, the otherArgs array is of size 0.
//Hence the output will be: a: 1 b: 3.
```

There a few key notes regarding rest parameters:

<ul>
<li> A function can only have 1 rest parameter. So `func(...firstParams, ...secondParams)` is invalid. </li>
<li> The rest parameter must be the last parameter in the function definition. </li>
</ul>

There are also some technical differences between rest parameters and the argument object:

<ul>
<li> The arguments object is not a real array, while rest parameters are Array instances, hence Array methods can be applied to it (sort(), map(), forEach(), pop(), etc)</li>
<li> The arguments object has additional functionality specific to itself like the callee property for example. </li>
<li> The ...restParam bundles all extra parameters into one array, therefore it doesn't contain any named argument defined before the ...restParam. On the other hand, arguments object contains ALL the parameters - including the parameters in ...restParam array - bundled into 1 array-like object.</li>
</ul>

One last note is that one can access the length of the rest parameters array: `function func1(...restParams1) { print(restParams1.length); }`

Overall, learning to use rest parameters in combination with default parameters allows very flexible, powerful and specific function definitions.
